## Name
EventDeck

## Purpose
Отображает активные карты событий кампании. Виджет получает готовый снапшот из движка (стопка, открытые карты, сброс) и выводит только то, что важно игроку: текущие активные события и счётчики карт в колоде и сбросе.

## Visual Overview
```
┌───────────────────────────────────────────────┐
│ КОЛОДА СОБЫТИЙ        [Колода  05] [Сброс 02] │
│ Активные события (2)                         │
│ ┌───────────────────────────────────────────┐ │
│ │ "Культист"      [Враг] [В игре]           │ │
│ │ Подозрительный человек...                │ │
│ │ Локация: Третья улица                    │ │
│ ├───────────────────────────────────────────┤ │
│ │ "Загустевшие тени"                      │ │
│ │ Ведущий выбирает карту в локации...      │ │
│ └───────────────────────────────────────────┘ │
└───────────────────────────────────────────────┘
```
- Фон: полупрозрачный тёмно-синий (rgba(15,23,42,0.82)).
- Карточки: градиент от #1e293b к #0f172a, левая рамка аквамаринового цвета.
- Чипы счётчиков повторяют стиль виджета руки (`CardHand`).
- Состояния: обычные карты, карты с бейджем «В игре» подсвечиваются красной рамкой.
- Скролл: вертикальный, с постоянным `scrollbar-gutter`.

## Behavior
- Виджет не предоставляет управляющих кнопок: движок сам решает, когда добирать карты.
- Список показывает только активные события. Если список пуст — выводится заглушка «Нет активных событий.»
- Счётчики «Колода» и «Сброс» отображают размеры стопки и сброса; при нуле чип становится полупрозрачным.
- Статусы от движка отправляются в скрытую `aria-live` область без визуального отображения.
- Карточки событий статичны: клики по ним ничего не делают.

## API (Props / Inputs / Outputs)
| Name | Type | Default | Description |
|------|------|---------|-------------|
| `EventDeck(container, config)` | `constructor` | — | Создаёт экземпляр и монтирует разметку в переданный контейнер. Конфигурация используется только движком для подготовки данных. |
| `applySnapshot(snapshot)` | `(EventDeckSnapshot) -> void` | — | Обновляет список активных событий и счётчики. |
| `setIntentHandlers(handlers)` | `(EventDeckIntentHandlers) -> void` | `{}` | Сохраняет колбэки, хотя виджет больше не вызывает их напрямую (сохранено для обратной совместимости). |
| `setStatus(message, variant?)` | `(string, EventDeckStatusVariant?) -> void` | — | Обновляет скрытый `aria-live` регион сообщениями от движка. |

## States and Examples
- **Пустой список**: нет карт в `revealed`, отображается заглушка.
- **Обычное событие**: карточка без бейджей, только заголовок и текст.
- **Враг в игре**: добавляются бейджи `Враг` и `В игре`, карточка получает красную рамку.
- **Большой список**: появляется вертикальная прокрутка, чипы остаются на месте.

## Lifecycle
- При создании виджет инжектит свои стили (одноразово на документ).
- Разметка не содержит внешних слушателей событий, поэтому при демонтаже достаточно удалить корневой элемент.
- Метод `setStatus` обновляет только скрытый `aria-live` регион и не требует очистки.

## Integration Example
```ts
import { EventDeck, type EventDeckConfig } from "./widgets/event-deck/event-deck";
import eventDeckSource from "./data/event-deck.json";

const container = document.getElementById("right-top");
const eventDeckConfig = eventDeckSource as EventDeckConfig;
const deck = new EventDeck(container, eventDeckConfig);

deck.applySnapshot({
    drawPile: [],
    revealed: [],
    discardPile: [],
});
```
